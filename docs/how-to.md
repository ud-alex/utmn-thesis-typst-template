
# Как использовать

## Ознакомьтесь с синтаксисом Typst

Здесь описаны только отдельные аспекты работы с Typst, которые специфичны для данного проекта. 

Прежде чем начать, ознакомьтесь с основами документации Typst на официальном сайте [https://typst.app/docs/](https://typst.app/docs/) или с иными туториалами.

## Структура проекта

Корневая директория шаблона содержит следующие ключевые элементы:

- `main.typ` – главный файл, в который подключаются все остальные части. Именно его нужно компилировать.
- `project_data.yml` – файл с персональными данными и параметрами нумерации (см. следующий раздел).
- `assets` - папка со вспомогательными файлами
  - `gost_citation_styles.txt` - Этот файл содержит ссылки на примерные библиографические стили по ГОСТу, которые можно скачать и использовать в проекте. Файл носит информационный характер и не участвует в компиляции. 
  - `appendix.typ` – вспомогательная функция для оформления приложений.
  - `bibliography.bib` – файл с библиографическими записями в формате BibTeX (создаётся пользователем).
  - `template.typ` – файл со стилями и общими настройками (поля, шрифты, нумерация). Редактируется только при необходимости глобальной смены оформления.
- `chapters/` – папка, содержащая все основные разделы работы:
  - `title_page.typ` – титульный лист (генерируется автоматически из YAML).
  - `intro.typ`, `first_chapter.typ`, `second_chapter.typ`, `conclusion.typ` – примеры стандартных глав.
  - `bibliography.typ` – список литературы (требует активации, см. раздел о библиографии).
  - `appendices.typ` – приложения (используют функцию `#appendix` из корневого `appendix.typ`).
- `demo.typ` – файл с примерами использования всех элементов (формулы, таблицы, рисунки, теоремы). Рекомендуется изучить его для быстрого освоения.
- `graphics/` – папка для изображений (PNG, JPG, SVG).

Все пути в `main.typ` уже настроены на подключение файлов из папки `chapters/`. При добавлении новых глав просто создайте файл в этой папке и добавьте строку `#include "chapters/новая_глава.typ"` в `main.typ` после остальных включений.

## Отредактируйте project_data.yml

Все ключевые метаданные вынесены в отдельный файл в корне проекта. Править исходный код вручную стоит только в исключительных случаях (некорректное отображение, необходимость кастомизации).

В этом файле заполните следующие поля:

- **Сведения об авторе**: `author`, `gender`, `year`, `university`, `city`, `institute`, `department`.
- **Данные о программе**: `major`, `major_code`, `submajor`, `education_mode`, `academic_year`, `education_level` (бакалавр / магистр / специалист).
- **Руководитель и завкафедрой**: `advisor`, `advisor_credentials`, `department_head`, `department_head_credentials`.
- **Настройки нумерации формул**:
  - `numbering_style`: `"continuous"` (сквозная) или `"nested"` (по главам).
  - `numbering_by_default`: `true` – нумеровать все центрированные формулы автоматически; `false` – только те, у которых есть метка (`<label>`).

После изменения YAML-файла перекомпилируйте проект – изменения применятся автоматически.


## Что нужно знать

Отбивать начало новой страницы нужно вручную в начале файла с главой. 

По умолчанию секции первого уровня не нумеруются (ВВЕДЕНИЕ, ЗАКЛЮЧЕНИЕ). Для создания полноценной главы вида (ГЛАВА 1. ТЕОРЕТИЧЕСКАЯ ЧАСТЬ) используйте тэг <core_chapter>:
`= Теоретическая часть <core_chapter>`

Названия глав и подглав отображаются прописными (заглавными) буквами автоматически.

Если Вы используете сторонний пакет, то импортировать его нужно в каждый файл, где он применяется.

Для добавления приложений в конце работы используйте функцию `#appendix` из файла `/appendix.typ`. Она сразу добавляет нумерацию в правом верхнем углу с заголовком посередине и индексирует его в «содержании».

Библиографические ссылки

1. Создайте файл bibliography.bib в корневой папке с записями в формате BibTeX (пример: @book{ключ, author="...", title="...", ...}).
2. Поместите CSL-файл с нужным стилем (например, ГОСТ Р 7.0.5-2008) в корень проекта.
3. В файле chapters/bibliography.typ раскомментируйте строку: `#bibliography("./bibliography.bib", title:"СПИСОК ИСТОЧНИКОВ", style: "/gost-r-7-0-5-2008-numeric-alphabetical.csl")`
4. В тексте цитируйте: @ключ (например, @commonsense). Список литературы автоматически сформируется в конце работы.

Примечание: не все CSL-файлы для ГОСТ корректно работают в Typst. Если оформление не соответствует требованиям, попробуйте другой стиль или настройте вручную.


## Настройка нумерации

### Нумерация формул

В шаблоне предусмотрена гибкая настройка нумерации **только для формул** через параметры в `project_data.yml`:

**1. `numbering_style` – стиль нумерации**
- `"continuous"` – сквозная нумерация по всей работе: `(1)`, `(2)`, ..., `(n)`.
- `"nested"` – нумерация внутри каждой главы: `(1.1)`, `(1.2)`, ..., `(2.1)`, `(2.2)`, ...

**2. `numbering_by_default` – автоматическая нумерация**
- `true` – все центрированные формулы нумеруются автоматически.
- `false` – нумеруются только формулы с меткой `<label>`.

| `numbering_style` | `numbering_by_default` | Результат |
|-------------------|------------------------|-----------|
| `"nested"` | `true` | Все формулы нумеруются как (1.1), (1.2), ... |
| `"nested"` | `false` | Нумеруются только формулы с меткой `<...>`; номера вида (1.1), (1.2), ... |
| `"continuous"` | `true` | Все формулы нумеруются как (1), (2), (3), ... |
| `"continuous"` | `false` | Нумеруются только формулы с меткой; номера вида (1), (2), (3), ... |

**Ручное управление:**

```typst
// Явно задать номер
#math.equation(block: true, numbering: "(A.1)")[ $E = mc^2$ ]

// Отключить нумерацию
#math.equation(block: true, numbering: none)[ $1+1=2$ ]

// Включить нумерацию через метку (если numbering_by_default: false)
#math.equation(block: true)[ $a^2+b^2=c^2$ ] <pifagor>
```

Компилятор может иногда ругаться на ручное подавление нумерации. Добавление любого тэга `<label_name>` должно устранить ошибку.

### Нумерация таблиц, рисунков и листингов

В отличие от формул, для этих элементов автоматическая настройка нумерации не предусмотрена, так как:

1. Они встречаются в дипломных работах реже.
2. В большинстве случаев достаточно сквозной нумерации по всей работе (Рисунок 1, Таблица 1, ...), которая реализована в шаблоне по умолчанию.
3. Настройка нумерации по главам для таблиц и рисунков потребовала бы усложнения кода, хотя и возможна при ручном редактировании template.typ.
